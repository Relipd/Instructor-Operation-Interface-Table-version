# 异常处理中心 — 回传逻辑文档

> 生成时间：2026-06-24 | 对应代码：`src/App.tsx`、`src/utils/bitable.ts`

---

## 一、整体架构

**单表设计**：数据表 = 回传表（同一张飞书多维表格）。所有状态变更、反馈、超时标记都写回主表本身，不再有独立的回传表/配置表。

```
用户操作 / 数据加载
        │
        ▼
  写入主表（updateRecordField / updateRecordFields）
        │
        ▼
  飞书 onDataChange 事件触发
        │
        ▼
  防抖 800ms → 重新拉取（fetchData）
```

## 二、三类回传场景

### 2.1 确认核查（待处理 → 处理中）

**触发**：用户在异常列表点击「确认核查」→ `ConfirmDialog` 选择指导员 → 确认

**代码位置**：`App.tsx` → `handleConfirmAction`

**写入字段**（同一次 `updateRecordFields` 批量写入）：

| 配置字段 key | 写入值 | 说明 |
|---|---|---|
| `statusFieldId` | `'处理中'` | 主状态推进 |
| `handlerFieldId` | 选择的指导员姓名 | 分配处理人 |
| `feedbackStatusFieldId` | `'处理中'` | 回传状态同步 |
| `feedbackTimeFieldId` | `Date.now()` | 操作时间戳 |
| `feedbackPersonFieldId` | 选择的指导员姓名 | 操作人 |

**本地状态**：立即 `handleRecordsBatchUpdate` 更新 `status='处理中'`、`handler`，UI 即时反馈，不等网络。

### 2.2 反馈处理（处理中 → 已完成）

**触发**：用户在异常列表点击「反馈」→ `FeedbackDialog` 填写反馈内容 + 处理人 → 提交

**代码位置**：`App.tsx` → `handleFeedbackSubmit`

**写入字段**（同一次 `updateRecordFields` 批量写入）：

| 配置字段 key | 写入值 | 说明 |
|---|---|---|
| `statusFieldId` | `'已完成'` | 主状态推进 |
| `feedbackResultFieldId` | 用户填写的反馈内容 | 反馈正文 |
| `feedbackTimeFieldId` | `Date.now()` | 反馈时间戳 |
| `feedbackPersonFieldId` | 处理人姓名 | 反馈人 |
| `feedbackStatusFieldId` | `'已反馈'` | 回传状态同步 |

**本地状态**：立即更新 `status='已完成'`、`feedback`、`handler`。`status='已完成'` 时 `calcTimeout` 返回 `false`，无需手动清除超时标记。

### 2.3 超时自动回传（加载时检测）

**触发**：`fetchData` 拉取主表数据后，遍历所有记录计算超时，对超时记录写回「是」。

**代码位置**：`App.tsx` → `fetchData`（数据拉取完成后）

**写入字段**：

| 配置字段 key | 写入值 | 说明 |
|---|---|---|
| `feedbackIsTimeoutFieldId` | `'是'` | 超时标记 |

**超时判定逻辑**（`calcTimeout`）：

```
status === '已完成'              → 不超时（已处理）
deadlineText 为空或 '-'          → 不超时（无时限）
reviewTime 为空                  → 不超时（无审阅时间）
deadlineText 含 '1'              → 时限 1 个工作日
deadlineText 含 '3'              → 时限 3 个工作日内
审阅时间 + 时限天数 < 当前时间    → 超时
```

**回传方式**：并发 5 路批量写入（`Promise.all`），避免串行阻塞。

## 三、关键防护机制

### 3.1 超时回传不阻塞显示

```
fetchData:
  ├─ 拉取主表 3k 行
  ├─ 计算超时标记（内存中）
  ├─ setAllRecords(...)        ← 立即显示（含 isTimeout 标记）
  ├─ setLoading(false)         ← 用户立即可交互
  └─ 后台：超时回传（异步）     ← 不阻塞显示
```

### 3.2 防 onDataChange 循环

**问题**：超时回传 `updateRecordField` 写入主表 → 触发飞书 `onDataChange` → 重新 `fetchData` → 又检测超时 → 又回传……死循环。

**解决**：`timeoutWriteRef` 标记

```
超时回传开始   → timeoutWriteRef.current = true
onDataChange   → 检测到 true → 跳过 fetchData（不循环）
回传完成       → 3 秒后 timeoutWriteRef.current = false（恢复）
```

**代码位置**：
- `App.tsx` `fetchData`：回传前检查 `!timeoutWriteRef.current`，回传时置 `true`，`setTimeout` 3 秒后置 `false`
- `App.tsx` `onDataChange` 监听：`if (timeoutWriteRef.current) return;`

### 3.3 onDataChange 防抖

短时间密集数据变更事件合并为一次拉取（800ms），避免重复全量读取。

```typescript
let timer = null;
const off = activeDashboard.onDataChange(() => {
  if (timeoutWriteRef.current) return;  // 回传自身触发，跳过
  if (timer) clearTimeout(timer);
  timer = setTimeout(() => fetchData(cfg), 800);
});
```

## 四、回传字段映射配置

所有回传字段在 `ConfigPanel` 的「反馈字段（与数据表同表）」分组配置：

| 配置面板标签 | 配置 key | 用途 |
|---|---|---|
| 异常记录编码（关联键） | `feedbackRecordCodeFieldId` | 关联键（当前同表，预留） |
| 回传状态 | `feedbackStatusFieldId` | 回传状态字段（处理中/已反馈） |
| 反馈结果 | `feedbackResultFieldId` | 反馈内容正文 |
| 反馈时间 | `feedbackTimeFieldId` | 操作时间戳 |
| 反馈人 | `feedbackPersonFieldId` | 操作人姓名 |
| 是否超时 | `feedbackIsTimeoutFieldId` | 超时标记（是/否） |

## 五、底层写入 API

`src/utils/bitable.ts`

### `updateRecordField(tableId, recordId, fieldId, value)`
单字段更新，调用 `table.setCellValue(fieldId, recordId, value)`。
**用于**：超时回传（单字段）。

### `updateRecordFields(tableId, recordId, fields)`
多字段批量更新，调用 `table.setRecord(recordId, { fields })`。
**用于**：确认核查、反馈处理（多字段一次性写入）。

## 六、数据流时序

### 确认核查 / 反馈处理

```
用户点击操作按钮
    │
    ▼
本地状态立即更新（UI 即时反馈）
    │
    ▼
updateRecordFields 写入主表（多字段）
    │
    ▼
飞书 onDataChange 触发
    │
    ▼
防抖 800ms → fetchData 重新拉取（验证写入成功）
```

### 超时自动回传

```
fetchData 拉取主表完成
    │
    ▼
遍历计算每条记录 isTimeout
    │
    ▼
setAllRecords（显示，含超时标记）
setLoading(false)
    │
    ▼
timeoutWriteRef = true（防循环）
    │
    ▼
并发 5 路写入超时记录的 feedbackIsTimeoutFieldId = '是'
    │
    ▼
写入触发 onDataChange → 被 timeoutWriteRef 拦截（跳过）
    │
    ▼
3 秒后 timeoutWriteRef = false（恢复正常）
```

## 七、注意事项

| 约束 | 说明 |
|---|---|
| **单表设计** | 数据表与回传表是同一张表，所有回传字段都在主表中 |
| **字段可选** | 回传时每个字段都判断 `if (config.xxxFieldId)`，未配置的字段跳过不写 |
| **超时回传幂等** | 已是「是」的记录不会重复写入（`if (timeout !== r.isTimeout)` 判断） |
| **防循环窗口** | 超时回传后 3 秒内的 onDataChange 被忽略，3 秒后恢复 |
| **并发限制** | 超时回传并发 5 路，避免压垮飞书 API |
